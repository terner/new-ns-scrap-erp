ตารางเทียบระบบ

สถานะนี้เป็น first pass สำหรับคุย scope ก่อน ยังไม่ใช่ final solution design

ระบบ เมนู/ขอบเขตในระบบเดิม ERPNext รองรับ สถานะ Priority คุยลูกค้า หมายเหตุ
ระบบกลาง / Admin Login, Users, Roles, Permissions, Company Profile, Audit Log มีพื้นฐาน user, role, permission, audit และ company setup Standard + Config คุยก่อน ต้องเทียบ role matrix เดิมกับ permission model ของ ERPNext ว่าละเอียดพอหรือไม่
Master Data ลูกค้า, ผู้ขาย, สินค้า, สาขา, คลัง, บัญชีเงิน, ช่องทาง, เครื่องจักร, currency มีเกือบครบใน ERP มาตรฐาน Fit คุยก่อน เป็นกลุ่มที่ย้ายเข้า ERPNext ได้ง่ายสุดและควรใช้เป็นฐานของระบบใหม่
ซื้อ (Procurement) บิลรับซื้อ, supplier flow, payment link มี purchase cycle มาตรฐาน Fit คุยก่อน ต้องเช็กเรื่องฟอร์มรับซื้อ scrap, น้ำหนัก, lot และเอกสารเฉพาะไทย
ขาย (Sales) บิลขาย, customer receipt flow, pricing มี sales order / invoice / receipt มาตรฐาน Fit คุยก่อน ต้องเช็กเรื่องต้นทุนต่อบรรทัด, pending sale และรูปแบบ document เฉพาะ
สต๊อกและคลัง Stock Balance, Ledger, Transfer, Warehouse มี stock master, warehouses, stock ledger, inventory reports Fit คุยก่อน เป็นแกนร่วมของซื้อ-ขาย-ผลิต ต้องพิสูจน์ว่า item, lot, movement types ตรง use case scrap แค่ไหน
AR / AP ลูกหนี้, เจ้าหนี้, aging, balances มีตรง Fit คุยก่อน มักใช้ของมาตรฐานได้ค่อนข้างตรงเมื่อซื้อขายและบัญชีลงตัว
ค่าใช้จ่าย / อนุมัติ / เงินสำรองจ่าย Expense, Payment Approval, Petty Advance, Returns มีบางส่วนผ่าน expense / payment / workflow Standard + Config คุยระยะ 2 ตัวเอกสารและ workflow อนุมัติอาจทำบน ERPNext workflow ได้ แต่ petty advance ต้องตรวจละเอียด
การผลิต Production Order, WIP, Yield, Machine Utilization, Production Cost มี manufacturing module Standard + Config คุยระยะ 2 ตัวผลิตมีฐานให้ แต่รายงาน yield/loss และ scrap-specific cost allocation อาจต้อง custom เพิ่ม
บัญชี / งบการเงิน / ภาษีไทย P&L, Balance Sheet, Cash Flow, VAT, WHT ERPNext + ERPNext Thailand มีพื้นฐานแข็งแรง Fit คุยก่อน นี่เป็นจุดแข็งของ ERPNext โดยเฉพาะถ้าลูกค้าต้องการมาตรฐานบัญชีและภาษีไทย
การเงินต่างประเทศ FX, FCD, overseas transfer/receipt, bank recon มีบางส่วน แต่ไม่ตรงทั้งหมด Standard + Config คุยระยะ 2 ต้องแยกว่าต้องการแค่ multi-currency หรือมี flow เอกสาร/charge เฉพาะธนาคารเพิ่ม
Trading Trading Dashboard, Trading Matching, PO Outstanding ไม่ตรงกับ standard module โดยตรง Custom คุยระยะ 2 มีแนวโน้มต้องทำ app/doctype/report เพิ่มบน ERPNext
Dual Costing PO Buy, PO Sell, Cost Pool, Cost Allocator, Match Log ไม่ใช่ logic ERP มาตรฐาน Custom คุยก่อน เป็นหนึ่งใน gap ใหญ่สุด และมีผลกับการตัดสินใจว่าจะไป ERPNext คุ้มไหม
Scrap-specific Workflow Grade Adjustment, Status Convert, Pending Sale, Receipt Voucher มีบางส่วนใกล้เคียง แต่ไม่ตรง Custom คุยก่อน เป็น business process เฉพาะของบริษัทและไม่ควร assume ว่าของมาตรฐานแทนได้
Dashboard / Tracking 360 / Anomaly Owner Daily, Tracking, Profit Leak, Anomaly Detector มี report builder และ dashboard พื้นฐาน Custom คุยหลัง มักทำหลังจาก core transaction และ data model นิ่งแล้ว
Offline-first localStorage, IndexedDB, browser-first sync ไม่ใช่แนวหลักของ ERPNext ต้องวิเคราะห์เพิ่ม คุยก่อน ถ้าลูกค้าต้องการ offline จริงจัง นี่อาจเป็นเงื่อนไขสำคัญที่ทำให้ต้องคิดใหม่ทั้ง platform

ข้อสรุปสำหรับคุยลูกค้า

ถ้าจะใช้ ERPNext แนวทางที่ pragmatic ที่สุดคือใช้ของมาตรฐานให้มาก แล้วทำเพิ่มเฉพาะ gap ที่เป็นธุรกิจ scrap จริง ๆ

โมดูลที่น่าจะใช้ ERPNext ได้คุ้ม

  • Master Data
  • ซื้อ / ขาย
  • สต๊อก / คลัง
  • AR / AP
  • บัญชี / งบการเงิน / ภาษีไทย
  • Assets พื้นฐาน

โมดูลที่ต้อง fit-gap ให้ละเอียดก่อนตัดสินใจ

  • Dual Costing
  • Trading Matching
  • Scrap-specific workflow
  • Production cost / yield reports
  • Offline-first requirement
