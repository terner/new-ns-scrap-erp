# แผนการปรับปรุงเวอร์ชัน Responsive Mobile และระบบ Typography ทั้งโครงการ (plan23-6-26)

เอกสารแผนงานนี้รวบรวมหน้าจอฝั่ง Client Component ทั้งหมด **56 ไฟล์** ในระบบ **NS Scrap ERP** เพื่อกำหนดแนวทางปรับปรุงเลย์เอาต์บนมือถือ (Mobile UI Rehabilitation) ให้เป็นไปในทิศทางเดียวกันทั้งหมดตามมาตรฐาน **AcexPOS UI Standard (Peach)**

---

## 🎯 วัตถุประสงค์หลัก
1. **เพิ่มความสะดวกในการอ่าน (High Legibility):** ขยายขนาดตัวอักษรบนหน้าจออุปกรณ์พกพา (Mobile Viewport) ให้เหมาะสมกับกลุ่มผู้ใช้งานในโรงงาน โดยมีขนาดข้อความรายละเอียดพื้นฐานไม่ต่ำกว่า 14px (`text-sm`)
2. **ขจัดตารางแบบเลื่อนขวา (No Horizontal Scroll):** เปลี่ยนทุกตารางแสดงผลหลัก (Desktop Tables) ให้กลายเป็นการ์ดแนวตั้ง (Mobile Card List View) เมื่อแสดงผลบนหน้าจอขนาดเล็กกว่า 1024px (`lg` breakpoint)
3. **ปรับการทำรายการให้เหมือน Mobile App จริง (Native App Feeling):** ลดความแออัดของปุ่มกดที่การ์ด โดยรวบปุ่มคำสั่งต่างๆ เข้าด้วยกันเป็นปุ่มเดียว "จัดการรายการ" (Manage Actions) ที่จะเปิด Dialog เมนูปุ่มสัมผัสขนาดใหญ่ (`h-12 text-base`) ที่กดง่ายและมีความคมชัดสูง

## 🎨 มาตรฐานการออกแบบ (UI Sizing & Structure Matrix)

| ส่วนประกอบ UI | ขนาดข้อความ (Tailwind Class) | ความสูง (Touch Target Height) | สีสันและสไตล์เพิ่มเติม |
| :--- | :--- | :--- | :--- |
| **หัวข้อหลักของการ์ด (Card Headers)** | `text-base` (16px) ถึง `text-lg` (18px) | Auto | ตัวหนา `font-bold text-slate-900` |
| **ป้ายกำกับฟิลด์ (Labels)** | `text-sm` (14px) | Auto | ตัวหนาปานกลาง `font-medium text-slate-500` |
| **ค่าข้อมูลในฟิลด์ (Values)** | `text-base` (16px) | Auto | ตัวหนา `font-semibold text-slate-900` |
| **ข้อความสรุป / ยอดรวม (Metrics)** | `text-lg` (18px) ถึง `text-xl` (20px) | Auto | ตัวหนาเด่นสีชัดเจนตามสัญญาณการเงิน |
| **ปุ่มหลักบนการ์ด (Manage Actions)** | `text-sm` (14px) / `text-base` (16px) | `h-10` ถึง `h-11` | ขอบ Slate-300 พื้นหลังขาว โฟกัส Ring บางเบา |
| **ปุ่มกดใน Dialog เมนู (Action Sheet)** | `text-base` (16px) | **`h-12`** | ไอคอนนำหน้า ปรับสีตามปุ่ม (ปกติสี Slate / ยกเลิกสีแดงพาสเทล) |
| **ป้ายสถานะ (Status Badges)** | `text-xs` (12px) ถึง `text-sm` (14px) | Auto (Padding `px-2.5 py-1`) | ขอบมนทรงกลม (Rounded-full) พร้อมจุดสีบ่งชี้สถานะ |
| **หัวข้อ Dialog (Modal Header)** | `text-base` (16px) ถึง `text-lg` (18px) | Auto | สีพื้นหลังเข้ม `bg-slate-900` และข้อความสีขาว `text-white` |

## 🛠️ แผนขั้นตอนการดำเนินงานทีละหน้า (Step-by-Step Execution Plan)

### ขั้นตอนที่ 1: จัดกลุ่มการแก้ไขแบ่งออกเป็น Batch
เราจะแบ่งงานแก้ไขโค้ดออกเป็นรอบๆ (Batch) ละ 1 โมดูลตามหัวข้อด้านบน เพื่อไม่ให้ไฟล์ที่ต้องรีวิวรวมกันเยอะเกินไป และช่วยให้ตรวจสอบความเข้ากันได้ของ Type ได้ดีที่สุด

### ขั้นตอนที่ 2: รื้อตารางจัดทำ Responsive Grid
1. เพิ่มคลาส `hidden lg:table` ในแท็ก `<table>` เพื่อซ่อนตารางบนจอเล็ก
2. สร้าง `<div className="block lg:hidden space-y-3">` เรนเดอร์การ์ดแสดงผลแนวตั้ง
3. *ข้อยกเว้นสำหรับงบบัญชีและงบดุล:* ในกรณีหน้างบบัญชีขนาดใหญ่ที่ไม่สามารถแปลงเป็นการ์ดแนวตั้งได้ ให้ใช้กล่องครอบครอบด้วย `overflow-x-auto` และกำหนดขนาดตัวหนังสือให้อ่านง่าย เพื่อให้ผู้ใช้สามารถเลื่อนตารางแนวนอนได้อย่างมีสัดส่วน

### ขั้นตอนที่ 3: ปรับขนาดฟอนต์ให้ตรงกับ Sizing Matrix
* ตรวจสอบส่วนประกอบรายละเอียดที่เคยใช้ `text-[10px]` หรือ `text-xs` ให้ขยายขึ้นมาเป็น `text-xs font-semibold` (ในกรณีป้ายสถานะ) หรือ `text-sm` (กรณีฟิลด์และค่าข้อมูลทั่วไป) เพื่อความสมบูรณ์ในการอ่าน

### ขั้นตอนที่ 4: รวบปุ่มคำสั่งไปที่ Action Dialog
* ในหน้าจอใดก็ตามที่มีเมนูปุ่มกดหลายปุ่มต่อแถว ให้ใช้การผูกสเตท `[activeActionsRow]` และแสดง Modal สัมผัสขนาดใหญ่แทนบนหน้าจอมือถือ

### ขั้นตอนที่ 5: การตรวจสอบและการตรวจสอบความถูกต้อง (Validation)
* ทุกครั้งหลังการแก้ไข 1 Batch จะต้องรันการทดสอบคอมไพล์ในเครื่องโลคอลเสร็จสิ้นก่อนรายงานผล:
  ```bash
  npm run type-check --workspace @ns-scrap-erp/next
  npm run lint --workspace @ns-scrap-erp/next
  npm run build --workspace @ns-scrap-erp/next
  ```
